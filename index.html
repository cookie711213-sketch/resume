ERP · Integrity

계산과 동시성의 경계 정리

Next.js · TypeScript · Firestore · Cloudflare

  • 반올림 기준을 하나의 계산 함수로 통합해 전자증빙과 원장의 1원 단위 불일치를 해소했습니다.
  • 오래된 캐시로 현금영수증 문서번호가 충돌하던 흐름을 서버 최종 채번과 중복 시 최대 3회 재시도로 변경했습니다.
  • Cloudflare Edge 상담 선점 경로를 Firestore REST updateTime CAS로 재설계해 하나의 요청만 확정되게 하고, 충돌·필드 유실·전체 문서 덮어쓰기 회귀를 방어했습니다.

CRM · Performance

운영 병목과 장애 범위 축소

Kotlin · Spring Boot · MariaDB · React · AWS

  • 정산 프로젝션과 30건 배치 조회로 초기 로딩을 1분 이상에서 수초로 단축하고 응답 전송량을 약 1/6로 줄였습니다.
  • CloudWatch 다섯 지표의 연쇄를 분석해 리소스를 조정하고 고비용 알림 조회를 차단하여 서비스를 긴급 복구했으며, 후속 개선 과제로 분리했습니다.
  • 기본 계약 목록 진입을 페이지당 30건의 Firestore 커서 쿼리로 바꿔 초기 읽기 범위를 제한했습니다.
